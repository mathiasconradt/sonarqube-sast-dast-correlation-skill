# Correlation Examples

This directory contains real-world examples of SAST-DAST correlation reports generated by the SonarQube SAST-DAST Correlation skill.

## Available Examples

### 1. SonarQube + StackHawk (`sonarqube-stackhawk-example/`)

Real correlation analysis of a Java Spring Boot application.

- **SAST**: SonarQube with SonarJava security rules
- **DAST**: StackHawk DAST scanner
- **Correlations**: 2 HIGH CONFIDENCE (SQL Injection, XSS)
- **SAST Issues**: 102 (20 Critical, 52 High, 28 Medium, 2 Low)
- **DAST Issues**: 49 (2 Critical, 6 High, 41 Low)

**Key Highlights**:
- Complete taint flow analysis from HTTP requests to vulnerable sinks
- Exact endpoint matching (POST / and GET /products/direct)
- Demonstrates typical 2% correlation rate
- Shows complementary coverage between SAST and DAST

[View Example →](sonarqube-stackhawk-example/)

### 2. SonarQube + ZAP (`sonarqube-zap-example/`)

Real correlation analysis using OWASP ZAP for DAST scanning.

- **SAST**: SonarQube with SonarJava security rules
- **DAST**: OWASP ZAP (Zed Attack Proxy)
- **Correlations**: 2 HIGH CONFIDENCE (SQL Injection, XSS)
- **SAST Issues**: 102 (20 Critical, 52 High, 28 Medium, 2 Low)
- **DAST Issues**: 42 (2 Critical, 16 High, 2 Medium, 22 Low)

**Key Highlights**:
- Shows ZAP-specific attack payloads (`input='`, `</h1><scrIpt>alert(1);</scRipt><h1>`)
- Demonstrates open-source DAST tool integration
- Detailed exploitation evidence (HTTP 500 errors, reflected payloads)
- Comparison-ready: same codebase as StackHawk example

## Understanding Correlation Rates

**Why are correlation rates low (2-5%)?**

This is **normal and expected**:

- **SAST**: Detects potential vulnerabilities in code (broad coverage)
- **DAST**: Confirms runtime exploitability (validation)
- **Correlation**: Proves a code issue is actually exploitable

**Example breakdown**:
- 100 SAST issues → Potential vulnerabilities in code
- 40 DAST issues → Runtime exploitable vulnerabilities
- 2 Correlations → **Confirmed exploitable code issues** (highest priority!)

The correlated issues are your **highest confidence vulnerabilities** because:
1. SAST found the vulnerable code pattern
2. DAST proved it's exploitable at runtime
3. Both tools independently confirmed the same issue

## Correlation Confidence Levels

### HIGH Confidence ✅

- Same vulnerability type (SQL injection, XSS, etc.)
- Exact endpoint match (POST / ↔ POST /)
- Complete taint flow alignment (source → sink)
- **Action**: Fix immediately

### MEDIUM Confidence ⚠️

- Same vulnerability type
- Partial endpoint match (controller name matches)
- Incomplete taint flow data
- **Action**: Investigate and prioritize

### LOW Confidence ℹ️

- Same vulnerability type only
- No endpoint correlation
- **Action**: Review manually

## Contributing Examples

We welcome contributions of correlation examples! To add an example:

1. **Run the skill** on your project
2. **Anonymize sensitive data**:
   - Remove internal URLs
   - Redact project names/keys
   - Remove proprietary code snippets
3. **Create a directory**: `examples/your-example-name/`
4. **Include files**:
   - `README.md` (example description)
   - `sast-dast-correlation-report.md` (the report)
   - `correlations.json` (correlation data)
5. **Update this README** with your example
6. **Submit a Pull Request**

### Anonymization Checklist

- [ ] Remove real company/project names
- [ ] Redact internal URLs and IPs
- [ ] Remove authentication tokens
- [ ] Generalize file paths if needed
- [ ] Keep vulnerability details intact (for educational value)
